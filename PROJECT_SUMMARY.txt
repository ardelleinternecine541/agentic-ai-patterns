CORE PATTERNS IMPLEMENTED
=================================================================================

1. ReAct Pattern (271 lines)
   - Reasoning + Acting loop for autonomous agents
   - Components: Thought, Action, Observation dataclasses
   - Features: Iteration tracking, history management, state machine
   - Classes: ReActAgent, AgentState

2. Tool-Use Gateway (252 lines)
   - Centralized tool registration and execution
   - Input validation with optional JSON schema
   - Tool execution history tracking
   - Decorator-based tool registration
   - Error handling with custom exceptions
   - Classes: ToolGateway, ToolMetadata, 3 custom exception types

3. Memory Management (362 lines)
   - Short-term memory: Conversation buffer with FIFO eviction
   - Long-term memory: Vector embedding storage with semantic search
   - Hybrid memory system combining both
   - Cosine similarity calculation
   - Features: Metadata support, timestamps, capacity management
   - Classes: ShortTermMemory, LongTermMemory, HybridMemory

4. Fallback & Recovery (267 lines)
   - Configurable fallback chains
   - Multiple retry strategies: exponential, linear, fixed, immediate
   - Automatic exponential backoff with max delay limits
   - Execution history tracking
   - Graceful degradation support
   - Classes: FallbackChain, RetryStrategy, FallbackStep

5. Multi-Agent Orchestration (344 lines)
   - Task distribution across specialized agents
   - Dependency management and topological sorting
   - Result aggregation
   - Task status tracking
   - Classes: AgentOrchestrator, Agent, Task, TaskStatus

=================================================================================
KEY FEATURES & CAPABILITIES
=================================================================================

ReActAgent:
  - Autonomous reasoning and action loops
  - Tool integration via ToolGateway
  - Configurable iteration limits
  - Complete execution history
  - State management (THINKING -> ACTING -> OBSERVING -> DONE)

ToolGateway:
  - Decorator-based tool registration
  - Input schema validation (optional)
  - Tool execution routing
  - Comprehensive error handling
  - Execution history with timestamps
  - Tool listing and description queries

Memory Systems:
  - ShortTermMemory: Fixed-size conversation buffer
  - LongTermMemory: Vector-based semantic storage
  - HybridMemory: Combined system for context + knowledge
  - Configurable capacity and retrieval limits
  - Timestamp tracking for all entries
  - Semantic similarity search

FallbackChain:
  - Sequential step execution with retry logic
  - Exponential backoff with configurable initial/max delays
  - Multiple retry strategies for different use cases
  - Comprehensive execution history
  - Graceful degradation across fallback steps

AgentOrchestrator:
  - Multi-agent task distribution
  - Dependency tracking and topological sorting
  - Task status management (PENDING -> RUNNING -> COMPLETED/FAILED)
  - Result aggregation across agents
  - Circular dependency detection
  - Agent capability declaration

=================================================================================
USAGE EXAMPLES
=================================================================================

Basic ReAct Agent:
  from agentic_patterns import ReActAgent, ToolGateway
  
  gateway = ToolGateway()
  @gateway.register_tool(description="Calculate")
  def calculate(expr: str): ...
  
  agent = ReActAgent(model="gpt-4", tool_gateway=gateway)
  result = agent.run("What is 2 + 2?")

Tool Gateway:
  gateway = ToolGateway()
  @gateway.register_tool(description="Add numbers")
  def add(a: int, b: int): return a + b
  
  result = gateway.execute_tool("add", {"a": 5, "b": 3})

Memory Systems:
  from agentic_patterns import ShortTermMemory, LongTermMemory
  
  short_memory = ShortTermMemory(max_turns=10)
  short_memory.add("user", "What is Python?")
  context = short_memory.get_context(num_turns=5)
  
  long_memory = LongTermMemory()
  long_memory.store("Python is a programming language")
  results = long_memory.retrieve("What is Python?", top_k=3)

Fallback Chain:
  from agentic_patterns import FallbackChain
  
  chain = FallbackChain()
  chain.add_step(primary_api, max_retries=3)
  chain.add_step(fallback_api, max_retries=2)
  result = chain.execute()

Multi-Agent Orchestration:
  from agentic_patterns import AgentOrchestrator, Agent
  
  orchestrator = AgentOrchestrator()
  orchestrator.register_agent(planner_agent)
  orchestrator.register_agent(executor_agent)
  
  orchestrator.add_task("planning", "Plan the task", "planner")
  orchestrator.add_task("execution", "Execute", "executor", ["planning"])
  
  results = orchestrator.execute_pipeline()
